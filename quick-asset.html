Quick Asset

Quick Asset is a collection of tools that make coding faster.

Main Feature

  • Create different types of scripts faster
  • Create Toast with just 1 command
  • Pool Management
  • Popup Management
  • Data Save Management
  • SO Management
  • Some utilities
  • Integrate some popular Design Patterns

How To Use

1. Create different types of scripts faster

- Create Component:
+ Create Script/Unity/Component
+ Create a Script that inherits MonoBehaviour
- Create Scriptable Object:
+ Create Script/Unity/Scriptable Object
+ Create a Script that inherits ScriptableObject
+ The generated script will be automatically added to the Asset Menu with the path "Game Datas/[Script Name]"
- Create Editor:
+ Create Script/Unity/Editor
+ Create a Script that inherits Editor
- Create Editor Window
+ Create Script/Unity/Editor Window
+ Create a Script that inherits EditorWindow
- Create C# class
+ Create Script/C# Core/Class
- Create C# abstract class
+ Create Script/C# Core/Abstract Class
- Create C# interface
+ Create Script/C# Core/Interface

2. Create Toast with just 1 command

- Add NotiManager prefab to first scene
- Use NotiManager.Instance.AddNoti(string content, Color colorText, NotiType type)
+ There are 2 NotiTypes: NotiType.StackUp, NotiType.FlyInCenter
+ Default NotiType is NotiType.StackUp

3. Pool Management (this is old version, the new version will be update soon)

- Create a GameObject, that should be parent of pool elements
- Add PopupManagement script to newly created GameObject
- Open PopupManagement script and scrool to last line
- Add a new Tag in Tag enum, it is to identify pool elements.
- In Inspector of PopupManagement script, select Tag field
- Add GameObject prefab to Prefab field
- Add default quantity for pool elements in Quantity
- Add parent of pool elements to Container field, if not the newly created GameObject will be default
- Coding:
+ Use PoolManager.Ins.GetObject(Tag tag) to get a transform of element, the element will be enable
+ Use PoolManager.Ins.BackToPool(Transform element, Tag tag) toreturn the element to the pool, the element will disable itself

4. Popup Management

- Create a Popup prefab, make sure that have a script inherits PopupUI with T is that script class type
- Add a large GameObject that wraps the UI content to the MainUI field for a slide-down effect (optional)
- Add small ui to UIEffect field for zoom in effect (optional)
- Now you just need to use the command: [Script Name].Show() to display the Popup
- Use Hide() to hide popup, use can add a callback, that callback will active when popup destroy

5. Data Save Management

- Example:
- Create a data class (Example: CurrencyData)
- Create a management class (Example: CurrencyDataManagement) inherits SessionPref with T is data class (Example: CurrencyData)
- Add get set data function
- Now you can use your data any where

6. SO Management

- Note: Only support non-duplicate SO.
- For example: You have SO Quest and you create 10 different Quests and each Quest is configured by each SO, these SOs will not be able to use Quick Asset support tools. Instead, create 1 SO Quest Container, which will be 1 SO that contains all Quests and it is unique
- Create a SO Script inherits BaseConfig with T is script class type
- Now you can use your so any where with GetConfig command

7. Some utilities

- UI Condition Controller
+ In this case, I have a item in shop, But I only want this item to show up when I have 100 gold
+ So I create ShowItemShop script inherits UIConditionController
+ In OnBeforeCheck function, we need to add conditions to display through the AddCondition
+ In AddListener, we will add Observer key, every time the Observer event with the corresponding key is trigger, the check will be triggered again.
+ Add script to a GameObject that is parent of item GameObject
+ Drag item GameObject to EligibleUI field
+ If you want to show another GameObject when the GameObject item is not eligible to be displayed, drag that GameObject into the NotEnoughtConditionUI field. But if you don't want to show any other alternative GameObject, choose HideOnNotEnoughtCondition
